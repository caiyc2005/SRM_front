# 接口清单

---

## 认证模块

### POST /api/login
登录认证

**请求体：**
```json
{
  "userCode": "admin",
  "password": "password"
}
```

**响应体：**
```json
{
  "success": true,
  "token": "xxx",
  "user": {
    "userName": "admin",
    "userCode": "admin"
  }
}
```

## 用户管理（/api/User）

### GET GetUsers
获取用户列表

**响应体：**
```json
{
  "success": true,
  "data": [
    {
      "id": "USR001",
      "userCode": "admin",
      "userName": "admin",
      "isDel": false,
      "memo": "系统管理员"
    }
  ]
}
```

### POST AddUser
新增用户

**请求体：**
```json
{
  "userCode": "zhangsan",
  "userName": "张三",
  "password": "123456",
  "memo": "备注信息"
}
```

**响应体：**
```json
{
  "success": true,
  "message": "用户添加成功"
}
```

### PUT UpdateUser
修改用户

**请求体：**
```json
{
  "id": "USR001",
  "userCode": "admin",
  "userName": "管理员",
  "password": "newpassword",
  "memo": "修改备注"
}
```
> password 可选，留空则不修改密码

**响应体：**
```json
{
  "success": true,
  "message": "用户修改成功"
}
```

### GET GetRoles

**响应体：**
```json
{

      "roleID": "R001",
      "roleName": "仓管员",
      "isDel": false,
      "memo": "仓库日常管理"
    }
  ]
}
```

### GET GetUserRoles
获取用户-角色关联列表

**响应体：**
```json

  "success": true,
  "data": [
    {
      "id": "UR001",

```

### POST AddUserToRole

**请求体：**
```json
{
  "userID": "USR001",
```

## 采购订单管理（/api/Orders）

### GET GetOrders

**查询参数：**
| 参数 | 类型 | 说明 |
|------|------|------|
| orderCode | string | 订单编号（模糊搜索） |
| supplierID | string | 供应商ID |
| status | string | 订单状态（0-5） |
| pageNum | number | 页码 |
| pageSize | number | 每页条数 |

**响应体：**
```json

  "data": [
    {
      "orderID": "1",
      "orderCode": "PO20240601000001",
      "supplierID": "S002",

      "materialCount": 2,
      "totalAmount": "580.00",
      "deliveryDate": "2024-07-15",
      "createTime": "2024-06-01 10:00:00",
      "deliveryNo": "",
      "materials": [
        {
          "index": 1,
          "materialCode": "M1011",
          "materialName": "物料1",
          "spec": "标准规格",
          "qty": 20,
          "unitPrice": 15.50,
          "amount": 310.00
```

## 状态码说明

| 状态 | 含义 |
|------|------|
| 0 | 待确认 |
| 1 | 已确认 |
| 2 | 待发货 |
| 3 | 已发货 |
| 4 | 已收货 |
| 5 | 已完成 |

## 统一响应格式

所有接口统一返回格式：

```json
{
  "success": true,
  "message": "操作成功",
  "data": []
}
```

- `success`：布尔值，true 表示成功，false 表示失败
- `message`：提示信息，失败时必填
- `data`：列表数据或单个对象，可选
